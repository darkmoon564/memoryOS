# Security policy

Do not report vulnerabilities in public issues. Send a private report to the
maintainers with reproduction steps, affected versions, and impact. Until a
dedicated security contact is published, use the repository owner's private
contact channel.

Never include API keys, database passwords, memory contents, or user data in a
report. MemoryOS deployments should expose database ports only to trusted local
or private networks and keep secrets in deployment-managed secret storage.
